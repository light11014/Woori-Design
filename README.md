금융앱에서 사용할만한 컴포넌트 설계/구현/배포

1. 금융 도메인에서 사용하는/할만한 컴포넌트 설계 및 구현

컴포넌트 설계/구현 시 고려할 부분

-   재사용성(컴포넌트를 설치해서 사용하는 실제 사용자 관점)
-   외부 모듈 의존도(컴포넌트를 유지보수하는 개발자 관점)

ex. 보안 키패드

-   숫자 위치 랜덤
-   숫자가 클릭될 때마다 랜덤으로 다른 버튼들도 클릭효과 처리하기
-   입력 시 화면에 표시될 실제 값은 마스킹 처리(● ● \_ \_ )
-   클립보드 복사 기능 차단

그 외

-   PIN/간편비밀번호 입력용 컴포넌트
-   OTP / 인증코드
-   계좌번호 or 금액 입력
-   거래 진행 상태 등
